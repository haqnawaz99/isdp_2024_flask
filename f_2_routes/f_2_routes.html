Introduction to Flask Routes

In Flask, a route is a URL pattern that the application recognizes. Each route is linked to a view function, which Flask executes when someone visits the associated URL. This is how Flask maps URLs to the functions that handle them.

How Routes Work

Routes are defined using the @app.route() decorator. The route decorator tells Flask which URL should trigger a particular function. The function then returns the content that will be displayed in the user’s browser.

For example:

@app.route('/')
def index():
    return "Hello World"

This code sets up a route for the root URL ('/'). When the user visits the home page of the Flask app, the message "Hello World" will be displayed.

Advanced Routes

Flask routes can also handle dynamic content by accepting parameters directly from the URL. You can define routes that process these parameters and return responses based on the input. Below are some examples of advanced routes:

1. The /salam Route

@app.route('/salam'): This route responds to the URL /salam. When a user visits this URL, the server responds with a greeting in Arabic, "السلام علیکم".

2. The /greet/<name> Route

@app.route('/greet/<name>'): This route is dynamic because it takes a variable <name> from the URL. Whatever name is provided in the URL, the function will return a personalized greeting using that name.

Example: If a user visits /greet/Ali, the response will be "السلام علیکم Ali". This demonstrates how Flask can use URL parameters to generate dynamic content.

3. The /square/<int:num> Route

@app.route('/square/<int:num>'): This route also takes a variable from the URL, but this time it's an integer <num>. The route calculates the square of the number provided and returns the result in a message.

Example: If a user visits /square/4, the response will be "The sum of 4 = 16". This shows how Flask can perform calculations based on input directly from the URL.

4. Running the Flask Application

The application is started with the same command as before:

app.run(host='0.0.0.0', debug=True)

This runs the Flask development server, making the routes accessible. The debug=True parameter ensures that the server provides detailed error messages and reloads automatically when changes are made to the code.

Conclusion

This session introduces the concept of routes in Flask and demonstrates how to create both static and dynamic routes. With these tools, you can build interactive web applications that respond to user input in various ways.
